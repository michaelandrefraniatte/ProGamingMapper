Program to Use All Controllers in All Games

The program allow to use All Controllers in All Games. You can remap input values of Physical Keyboards, Mices, Airmouse, Microphone Speech, Webcam with IR led, Xbox Controllers, PPJoy Controllers, Joycon left, Joycon right, Joycon Charging Grip, Pro Controller, Wiimote, Nunchuck, Sensor Bar, DualShock4 Controller, and DualSense Controller, to output values of Virtual Keyboard, Mouse, and Xbox Controllers. Writing some Scripts and it goes. It's easy to change key bindings and set some macros to help you in games. It's easy to find the best settings in order to control perfectly your character in games. Share some Scripts with the other users or even sale it, with the encrypted version of your Scripts, by choosing yourself who can run it. There isn't any input lag, nor input delay, nor stick drift. It's simple to use with everything already prepared, colored and completed, but you can play the way you want by writing some Scripts to more things with no limit. The program is easy to use with autocompletion and colors of inputs and outputs. You can check everything, All the inputs and outputs. Everything is already well managed, and also All the Possible manners to play with available Scripts. Let's see...

Considerations About the Program

I can give you some Script examples and helping you. Theres's a driver of old generation to install manually under peripheral manager from control panel (choosing the inf extension file), and another to reorder All your Physical and Virtual Controllers from joy.cpl (Win + R) if needed. Code Example Scripts

It's simple as that, inputs to outputs. Here the directionnal cruce of Wiimote send WASD Keyboard keys. It's very simple and stupid example, but you will find far better ways for moving your character or racing car in your games.

SendW = WiimoteButtonStateUp;
SendA = WiimoteButtonStateLeft;
SendS = WiimoteButtonStateDown;
SendD = WiimoteButtonStateRight;

It's just easy as that for sale your encrypted script. Each time you save your script, 2 files are created, 1 is your script codes, 1 is your script sales. Don't use this line code if you want all users can play with your script. You can add one to several users.

UsersAllowedList = new string[] { "username1", "username2" };

There's some inner usefull variables to have the control like for switching of configurations, or for automatically set an output later after another one.

/* Set to false getstate */
getstate = false;
/* Initialize pollcount */
pollcount = 0;

There's some variables to adapt the outputs for your prefered rendering, like recenter Wiimote IR for the center of your screen in function of sensor bar at top or bottom of your screen. Also like adapt speed of mouse move sent lowering its sensitivity with poll time.

/* Recenter Wiimote IR */
centery = -200;
/* Set the sleep time to wait between each poll (must be set between 1 and 50 ms) */
sleeptime = 20;

Here's some codes to set mouse cursor position from gyroscope axis of right Joycon.

/* Set mouse cursor axis X from Joycon right gyro axis X */
MouseDesktopX = width / 2f - JoyconRightGyroX * width / 2f / 1024f;
/* Set mouse cursor axis Y from Joycon right gyro axis Y */
MouseDesktopY = height / 2f + JoyconRightGyroY * height / 2f / 1024f;

There's an integrate function to rescale values. It's usefull for remove deadzone or for change easily sensistivity like in the following example.

/* Scale and set values of mouse move axis X from Wiimote IR axis X */
MouseMoveX = Scale(irx, -1024, 1024, -127, 127);
/* Scale and set values of mouse move axis Y from Wiimote IR axis Y */
MouseMoveY = Scale(iry, -1024, 1024, -127, 127);

Here the codes to assignate Joycon right stick axis to Xbox Controller right stick axis when Joycon right is used alone.

if (JoyconRightStickY > 0.25f) controller1_send_rightstickx = 32767;
if (JoyconRightStickY < -0.25f) controller1_send_rightstickx = -32767;
if (JoyconRightStickY <= 0.25f & JoyconRightStickY >= -0.25f) controller1_send_rightstickx = 0;

And the codes to assignate Nunchuck stick axis to Xbox Controller stick axis, almost similar.

if (WiimoteNunchuckStateRawJoystickX > 33f) controller1_send_rightstickx = 32767;
if (WiimoteNunchuckStateRawJoystickX < -33f) controller1_send_rightstickx = -32767;
if (WiimoteNunchuckStateRawJoystickX <= 33f & WiimoteNunchuckStateRawJoystickX >= -33f) controller1_send_rightstickx = 0;

The codes to set Xbox Controller left stick axis from Mouse axis is as following if the deadzone of Xbox Controller left stick in game options is set to 0.

/* Set Xbox Controller left stick axis X from Mouse axis X */
controller1_send_leftstickx = MouseHookX / 1024f * 32767f;
/* Set Xbox Controller left stick axis Y from Mouse axis Y */
controller1_send_leftsticky = -MouseHookY / 1024f * 32767f;

The codes to detect if a value changes, corresponding to the difference before and after, with a simple example (can be used to have a true ratchet control moving left at left and moving right at right but nothing when returning to center, but it need to use pollcount for detecting motion).

/* Detect the mouse move */
ValueChange[0] = (double)MouseHookX;
/* Amplify the mouse move */
MouseMoveX = _ValueChange[0];

Here's the sample for Microphone Speech.

/* Set the speech to recognize */
SpeechToText = new string[] { "Load", "Blow", "Fire", "Aiming", "Welcome", "Hello" };
/* Send Keyboard key R if you say load */
SendR = TextFromSpeech == "Load" ? true : false;

The outputs can be shared over internet using another program named PGMHost to run on PC you want to control. PGM and PGMHost need the IP and port of PC which control with PGM the PC you want to control running PGMHost. For streaming your PC, it must run RemoteGameplayHost which need the IP, the display port, the audio port, and the screen size of PC which control running RemoteGameplay, to have the gameplay and audio of the PC you want to control. There's a params.txt file for RemoteGameplay, to set IP and ports of PC you stream the gameplay and audio (PC you want to control), under same folder of the program.

NetworkDescriptor = new string[] { "192.168.1.10", "62000", "Home Network Lan" };

Help and Tips

• Input Devices: Mouse, Keyboard, XInput Controller, DirectInput Controller, Joycons as DirectInput Controller, Pro Controller as DirectInput Controller over Bluetooth, Dualshock4 Controller as DirectInput Controller, Dualsense Controller as DirectInput Controller, Wiimote and Sensor Bar and Nunchuck, Joycon left, Joycon right, Joycon Charging Grip over USB, Pro Controller over USB, Dualshock4 Controller over USB, Dualsense Controller over USB, Webcam and Led, Voice Speech to Text.
• Output Devices: Keyboard, Mouse, Xbox Controller.
• Pairing: Use the dedicated buttons for Bluetooth Controllers or use USB wire.
• Data: Script must running to see data values changing.
• Keyboard Layout: In game options, change key assignation, to remember easy which key to press.
• Mouse Sensitivity: In game options, set mouse sensitivity to lowest as possible, for playing FPS having a fine aim in center, but still turn fast.
• Setting: Tweak your scripts until find your best comfort.
• Limitation: Take care to wait before starting again script process after stopping it, because of Controller closing/activation.
• Caution: Don't use a Wii or candles or Mayflash to task as IR source for the Wiimote but a USB Wii U sensor bar.
• Trick: In top down shooters or rail shooters, if there isn't any option to enable desktop cursor, you can try to set the game in windowed mode, and remove the border if there isn't any option for windowed borderless mode, with WindowTitleRemover, in order to make running desktop cursor.
